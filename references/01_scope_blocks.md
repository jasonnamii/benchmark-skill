# 스코프별 구조 블록 (B2 상세)

4대 스코프별로 [B2] 구조 블록의 하위 항목이 달라진다. 나머지 4블록(B1·B3·B4·B5)은 공통.

---

## A. 조직·서비스·BM (HBR 예시형)

**B2 하위 4분할**

| 항목 | 내용 |
|---|---|
| 조직 | 모체·법인형태·핵심 부서 3~4개 |
| 서비스 스택 | B2C / B2B / 지식상품 등 레이어별 |
| BM | ①구독 ②B2B교육 ③콘텐츠판매 ④라이선싱 식 번호 나열 |
| 규모 | 독자·매출·브랜드파워 (추정은 확신도 낮게) |

**예: HBR**
```
조직: Harvard Business School 산하 비영리 / Harvard Business Publishing 운영
       Editorial·Product·Sales&Marketing·Corporate Learning 4부서
서비스: B2C(매거진·웹·뉴스레터) / B2B(기업교육·라이선스) / 지식상품(북·케이스)
BM: ①구독 ②B2B교육(핵심) ③콘텐츠 ④라이선싱
규모: 수백만 독자, 포춘500 다수, 매출 수천억 추정(비상장·낮음60)
```

---

## B. 제품·기능·UX

**B2 하위 4분할**

| 항목 | 내용 |
|---|---|
| 핵심기능 | 경쟁사와 다른 3~5개 기능만 |
| UX 패턴 | 온보딩·핵심 인터랙션·락인 구조 |
| 기술스택 | 공개된 범위만 (프론트·백·AI·인프라) |
| 성장 레버 | 바이럴·리텐션·수익화 중 작동하는 것 |

**예: Notion**
```
핵심기능: 블록 에디터 / 데이터베이스 / 템플릿 갤러리 / 협업
UX: 빈 페이지→템플릿 유도 온보딩 / 드래그블록 인터랙션 / 워크스페이스 락인
기술: React 프론트 / 자체 블록DB / AI 추가(OpenAI 기반)
성장: 템플릿 바이럴 + 개인→팀 전환(freemium) + 에듀 무료
```

## C. 캠페인·마케팅·카피

**B2 하위 4분할**

| 항목 | 내용 |
|---|---|
| 메시지 | 1문장 메인 카피 + 서브 카피 |
| 채널 | TVCF·디지털·OOH·바이럴 등 주 채널 |
| 크리에이티브 | 비주얼 스타일·음악·연출 특징 |
| 결과지표 | 인지도·매출·수상 등 공개 수치 |

**예: Apple "1984"**
```
메시지: "1984 won't be like 1984" — 빅브라더 저항 서사
채널: 슈퍼볼 TVCF 단일 게재 (1회 방영)
크리에이티브: Ridley Scott 연출 / 디스토피아 세트 / 망치 여성 히어로
결과: 3개월 Mac 1.5억$ 매출 / 광고사 명예의 전당 / 확신도 높음(90)
```

## D. 정책·제도·사회현상

**B2 하위 4분할**

| 항목 | 내용 |
|---|---|
| 배경 | 왜 생겼나 (사회·경제·정치적 원인) |
| 제도설계 | 핵심 메커니즘·거버넌스·재원 |
| 이해관계자 | 누가 이득·누가 반발 |
| 결과 | 지표·사이드이펙트·수명 |

**예: 싱가포르 HDB (공공주택)**
```
배경: 1960년대 슬럼·인종갈등 / 자가소유=국민통합 전략
설계: CPF(강제저축) 대출 / 95% 공공주택 공급 / 인종쿼터(민족 섞기)
이해: 정부(정권안정)·시민(자산)·이민자(통합) / 민간건설사 반발 흡수
결과: 자가소유율 90% / 주거안정 / 시장왜곡·노령화 대응과제 (높음85)
```

---

## 스코프 혼합

대상이 여러 스코프 걸침 (예: 스타벅스 = A+B+C) → **주 스코프 1개** 선정하고 나머지는 1줄씩 부가.

---

## Gotchas

| 함정 | 대응 |
|---|---|
| 4개 항목 모두 채우려 억지 | 모르면 "모름(30)" 선언. 빈칸 허용 |
| 스코프 D에서 이념 편향 | 이해관계자 블록에 양쪽 병치 |
| 스코프 B에서 기능 나열만 | UX·성장 레버가 더 중요. 기능은 3~5개만 |
| 스코프 C에서 수상만 강조 | 결과지표는 매출·인지도 같은 비즈 지표 우선 |
