# 출력 템플릿 (HBR스타일)

모든 벤치마킹 산출물은 이 템플릿을 따른다. 길이·구조 고정.

---

## SINGLE 모드 템플릿

```markdown
# {대상명} 벤치마킹

**스코프:** {A/B/C/D}  **모드:** SINGLE

---

## 1) 정의 + 성공/실패 이유
{대상 정의 1~2줄}
- 성공 이유: {1~2개}
- 실패·한계: {1~2개, 있으면}

## 2) 구조
{스코프별 4분할 — 불릿 4~6개}
- {항목1}: {1줄}
- {항목2}: {1줄}
- {항목3}: {1줄}
- {항목4}: {1줄}

## 3) WHY (왜 사람들이 쓰는가·사는가)
{대상} = "{인간 메커니즘 어구} — {1문장 설명}"
{필요 시 1~2줄 추가}

## 4) 차용 포인트 (우리가 가져올 것)
- {포인트1 — 기획에 즉시 적용 가능한 형태로}
- {포인트2}
- {포인트3, 선택}

## 5) 백본
→ **{대상} = "{핵심 정체성 어구}를 {작동 메커니즘}으로 {결과} 모델"**

**확신도:** {높음90 / 보통70 / 낮음50 / 모름30}
**근거:** {1줄}
```

---

## MULTI 모드 템플릿

```markdown
# {대상1} vs {대상2} vs {대상3} 벤치마킹

**스코프:** {공통 또는 혼합}  **모드:** MULTI

---

## 대상별 압축 5블록

### {대상1}
- 정의+성공실패: {2줄}
- 구조: {2줄}
- WHY: {1줄}
- 차용: {1줄}
- 백본: {1줄}

### {대상2}
{동일}

### {대상3}
{동일}

---

## 비교축 표

| 축 | {대상1} | {대상2} | {대상3} |
|---|---|---|---|
| {축1} | | | |
| {축2} | | | |
| {축3} | | | |

---

## 통합 차용 포인트
- 공통 패턴: {1~2줄}
- 차별화 축: {1~2줄}
- 우리 기획 적용: {1~2줄}

---

## 백본 (공통 or 대조)
→ **{공통 1줄} or {대조 2줄}**

**확신도:** {}
**근거:** {}
```

## METAPHOR 모드 템플릿

```markdown
# {원도메인} → {타겟도메인} 은유 벤치마킹

**스코프:** 크로스  **모드:** METAPHOR

---

## [원] {원도메인 대상} 5블록 요약
{SINGLE 모드 축약본, 블록당 2줄}

## 핵심 메커니즘 추출
- {메커니즘1}
- {메커니즘2}

---

## [매핑] {타겟도메인}에 적용
| 원 메커니즘 | 타겟 적용 |
|---|---|
| {메커니즘1} | {타겟 형태} |
| {메커니즘2} | {타겟 형태} |

---

## [충돌] 구조적 차이
- {충돌1}
- {충돌2}
- {충돌3, 선택}

---

## [변형] 충돌 우회안
- {변형1}
- {변형2}

---

## [판정]
- 차용 가능: {}
- 차용 불가: {}

**확신도:** {}
**근거:** {}
```

## 파일명 규칙

| 모드 | 파일명 |
|---|---|
| SINGLE | `{대상명}_bench.md` |
| MULTI | `{대상1}vs{대상2}_bench.md` (2개) / `{카테고리}_multi_bench.md` (3개+) |
| METAPHOR | `{원}to{타겟}_metaphor.md` |

옵시디언 볼트 저장 시: `02_Resources/benchmarks/` 폴더.

---

## Gotchas

| 함정 | 대응 |
|---|---|
| 템플릿 섹션 삭제 | 5블록 + 백본 + 확신도 필수. 빈 섹션이면 "모름" 채움 |
| 확신도·근거 누락 | UP v37 규칙. 무조건 병기 |
| 마크다운 헤더 레벨 틀림 | H1=대상명, H2=5블록, H3=하위 필요시만 |
| 옵시디언 wikilink 필요 | `obsidian-markdown` 스킬 협조 요청 |
