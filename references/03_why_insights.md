# WHY 블록 작성 가이드 (B3)

"왜 사람들이 이걸 쓰는가·사는가"를 **기능 나열**이 아니라 **인간 메커니즘**으로 쓰는 법.

---

## 원칙

기능을 쓰지 말고, **인간이 충족하려는 욕구**를 찾아라.

| 나쁜 예 (기능) | 좋은 예 (메커니즘) |
|---|---|
| "HBR은 퀄리티 높은 글이 많다" | "HBR은 지식 신분재 — 읽는다는 사실 자체가 계급 신호" |
| "스타벅스는 원두가 좋다" | "스타벅스는 집·사무실 사이 제3의 의식 공간" |
| "아이폰은 디자인이 예쁘다" | "아이폰은 선택 부담을 줄여주는 감각적 권위" |

---

## 12대 인간 메커니즘 (WHY 프레임)

**인지·심리**
1. **신분재·과시재** — 소유·소비 자체가 계급 신호 (럭셔리, HBR, 테슬라 초기)
2. **대체불가 서사** — 유일한 이야기 (애플 1984, 파타고니아 환경)
3. **감각적 권위** — 감각이 전문성을 보증 (다이슨, 러쉬, 블루보틀)
4. **선택 부담 제거** — 결정 피로 해결 (Apple, MUJI, Netflix 추천)

**사회·관계**
5. **정체성 표현** — 나의 부족·가치관 선언 (나이키, 파타고니아, 애플 스티커)
6. **소속감·부족주의** — 같은 브랜드 쓰는 사람과 연결 (애플 생태계, CrossFit)
7. **상호작용 유발** — 대화 시작 매개체 (에어팟 초기, 테슬라)

**진화·생존**
8. **안전·확실성** — 불안 해소 (보험, 애플 제품, IBM)
9. **시간·에너지 절약** — 생존자원 절약 (배달앱, Amazon Prime)

**감정·동기**
10. **자기효능감** — "나 이거 해냈다" (Duolingo, Peloton, Fitbit)
11. **소유의 기쁨** — 물질 이상의 애착 (레고, 다마고치, 애플)
12. **의식·리추얼** — 반복 행위의 안정감 (스벅 모닝커피, 매거진 구독)

---

## WHY 작성 공식

```
"{대상}은 {인간 메커니즘 어구} — {1문장 설명}"
```

**예시:**
- `HBR = "지식 신분재 — 읽는 것 자체가 커리어 시그널"`
- `스타벅스 = "제3의 의식 공간 — 집·사무실 사이 데일리 리추얼"`
- `Duolingo = "자기효능감 주사 — 매일 초록 불꽃이 성장 증거"`
- `Apple = "선택부담 제거 + 감각적 권위 — '애플을 샀다'로 결정 종료"`

## 다층 WHY (상위 제품일수록)

프리미엄·브랜드·사회현상일수록 **2~3개 메커니즘 결합**.

```
Patagonia = "정체성 표현(환경가치) + 대체불가 서사(1% for Planet) + 신분재(알릭 아웃도어)"
테슬라 = "대체불가 서사(지구 구원) + 상호작용 유발(자동차=대화주제) + 신분재(얼리어답터)"
```

## Gotchas

| 함정 | 대응 |
|---|---|
| 기능·스펙 나열로 빠짐 | 12대 메커니즘 중 최소 1개 명시 사용 |
| 너무 추상적 ("행복을 준다") | 구체 메커니즘 어구로 교체 |
| 여러 메커니즘 뭉뚱그림 | 메인 1개 + 서브 1~2개 계층화 |
| 카테고리마다 같은 WHY | WHY가 같으면 차별화 실패. 다른 메커니즘 찾기 |
| 우리가 좋아하는 것만 씀 | 안 쓰는 사람 관점 "왜 안 쓰나"도 1줄 고려 |

## 연계

- **hit-skill**: WHY 메커니즘이 히트 패턴 3층 중 "인간메커니즘 층"과 직결. 복잡한 경우 hit-skill 호출
- **copywriting-engine**: WHY 메커니즘이 카피 메시지 결정
- **human-skill**: 16축 메타원리로 WHY를 더 정밀 진단 가능
