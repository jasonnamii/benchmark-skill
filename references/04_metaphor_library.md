# 은유 벤치마킹 카탈로그

METAPHOR 모드에서 참조하는 "타 도메인에서 빌려올 성공 메커니즘" 카탈로그.

---

## 사용법

1. 타겟 도메인의 문제 파악 ("은행 지점이 죽어간다")
2. 아래 카탈로그에서 해결 메커니즘 보유한 원 도메인 검색
3. 매핑 + 충돌지점 검토 (`→ 02_mode_protocols.md METAPHOR`)

## 은유 매핑 체크리스트

은유 적용 전 반드시 확인:

- [ ] 원 도메인 핵심 메커니즘 1~2개로 정리했나 (너무 많으면 매핑 안 됨)
- [ ] 타겟 도메인에 **구조적 유사성**이 있나 (표면 유사 아님)
- [ ] 충돌지점 최소 2개 찾았나
- [ ] 충돌지점이 넘을 수 있는 수준인가 (못 넘으면 은유 폐기)
- [ ] 차용 가능·불가 구분했나

## Gotchas

| 함정 | 대응 |
|---|---|
| 표면 유사로 은유 ("둘 다 빨갛다") | 구조·메커니즘 유사성만 채택 |
| 충돌지점 무시 | 최소 2개. 무시하면 모조품 생산 |
| 은유 남발 (다 스벅처럼) | 도메인당 은유 1~2개. 과다는 혼란 |
| 카탈로그 외 대상 | 직접 추가 OK. 메커니즘만 분명하면 |
